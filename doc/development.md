# 开发与版本控制

当前版本采用 **0.2.x 自动递增**（基于 Git commit 数计算 patch），并保持主版本为 `0`，不升级到 `1.0`。

## 提交流程

本项目按“重要节点一提交”推进，每个里程碑独立 commit，例如：

1. 项目更名与目录迁移
2. 异步运行时与进程管道
3. 虚拟文本与浮动列表
4. AST 查询与性能优化
5. Layout Manager 与可插拔 UI 特性
6. PieceTable / 预测性渲染 / 终端能力降级链路

## 建议规范

- 提交主题聚焦单一目标
- 提交前保证可编译运行
- 复杂功能先 cProfile 定位热点再优化
